# How-to guides

How-to guides are task-oriented recipes. Use them when you already know what you
want to accomplish and need the shortest reliable path.

## Available guides

- [Run unsafe-review on a pull request](run-pr-review.md): inspect the current
  branch diff, choose output formats, and save artifacts for reviewers or CI.

## How-to rules

How-to guides should:

- start with prerequisites and a target outcome;
- prefer direct commands over background theory;
- include verification commands or expected artifacts;
- link to reference docs for exhaustive details and explanation docs for rationale.

They should not teach every concept from first principles.
