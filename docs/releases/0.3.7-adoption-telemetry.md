# 0.3.7 Adoption, Telemetry, and Real-Code Low-Noise Improvements

`unsafe-review` provides baseline-aware unsafe evidence coverage artifacts for
repo badges, PR gates, line-comment plans, LSP/LLM context, and ub-review.

0.3.7 — adoption, telemetry, and real-code low-noise improvements. It makes
`unsafe-review` easy to adopt and measured in use, tightens how findings are
framed and ranked on real code, and delivers a batch of evidence-discipline
correctness fixes across all output surfaces. It ships the post-0.3.6
adoption/telemetry lane plus a stance/projection correctness program validated
on real crates. It adds no analyzer breadth and no new claim: it remains
advisory static unsafe-coverage evidence.

## What changed

### Adoption surface

- **Composite GitHub Action** (`unsafe-review-first-pr`) for two-line CI
  adoption: wraps `first-pr`, uploads the review kit, appends the step
  summary, and exposes the gate manifest. Advisory by default — no automatic
  comment-posting, no blocking, no inherited-fail. Includes `fetch_depth`
  input, version-skew hardening, and live-smoke validation.
  ([#1628](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1628),
  [#1660](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1660),
  [#1661](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1661),
  [#1662](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1662),
  [#1663](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1663),
  [#1664](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1664))

- **`unsafe-review pr`** zero-config subcommand with repo/base autodetect.
  No silent full-repo scan and no ambiguous default blocking.
  ([#1629](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1629))

- Adoption front-door **`docs/START-HERE.md`** and outward-facing docs
  including a real-world dogfood narrative and agent-integration guide.
  ([#1635](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1635),
  [#1669](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1669))

- **External-first peak-RSS posture** ratified: ADR-0008 amended so resource
  measurement lives on the scheduled/bench path, not in-tool. Scheduled
  full-corpus backstop and external RSS harness (SPEC-0039).
  ([#1627](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1627),
  [#1631](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1631))

### Telemetry and measurement

- **Usefulness telemetry** projected from `ReviewCard`: cards/PR,
  new/worsened/resolved/inherited, selected vs not-selected reasons,
  agent-ready vs human-only, `scan_cost` (elapsed_ms/output_bytes),
  not-selected-class histogram, unfulfilled-obligation count.
  Telemetry is a usage/subset signal — not calibrated precision or recall.
  ([#1630](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1630),
  [#1634](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1634),
  [#1647](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1647))

### Framing and evidence discipline (stance/projection program #1705–#1718)

The stance/projection program corrects systematic output framing errors found
during real-code validation. These are not new detections: they are consistency
fixes so every surface projects from the same `ReviewCard` truth object with
evidence-disciplined framing.

- **SARIF level and LSP severity** derived from card class (single-source
  truth); no LSP severity-1 for low-class cards; SARIF rule description
  class-aware.
  ([#1706](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1706),
  [#1713](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1713))
- **`unsafe-review-gate.json`** emitted from `repo` mode (SPEC-0034 parity).
  ([#1705](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1705))
- **Receipt formal-tool provenance**: `site_reached` strength requires tool
  provenance; receipt records tool name and input hash.
  ([#1707](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1707))
- **Outcome markdown** includes unchanged cards for JSON parity.
  ([#1708](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1708))
- **Test reach** requires call-shaped owner use; self-reach rejected.
  ([#1709](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1709))
- **Owner cards grouped in human surfaces** while kept in artifacts and counts.
  ([#1710](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1710))
- **`debug_assert` does not discharge runtime guard obligations**: stripped in
  release builds, so it is not a runtime guard.
  ([#1715](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1715))
- **Per-card coverage-movement projection** from the summary.
  ([#1716](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1716))
- **`unsafe-review+` badge baseline-movement-aware**: reflects resolved/worsened
  delta.
  ([#1717](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1717))
- **`summary.unsafe_sites`** is now the real pre-cap site count.
  ([#1718](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1718))

### Truthfulness and selection fixes

- `agent_lsp_readiness` and agent-packet readiness derived from card class +
  `RequiresWitnessReceipt`: all surfaces agree on agent readiness.
  ([#1633](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1633))
- Comment-plan importance ranking: top-N by priority → gap-severity →
  confidence → file/line; bounded comment body (220 words, single-sourced).
  ([#1645](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1645),
  [#1646](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1646))
- `pub(crate)`/`pub(super)` visibility miscategorization fixed.
  ([#1666](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1666))
- Spread-aware card selection: `--max-cards` cap no longer blinds whole
  subsystems.
  ([#1667](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1667))
- Bare-name unsafe-op detectors gated on `unsafe { }` scope.
- `&mut *expr` / `&*expr` classified as `raw_pointer_deref`.
- `fn zeroed()` definition no longer carded as a zeroed call.
- Raw-pointer alignment/null guards anchored to receiver and position.
- `set_len` discharge requires joined `new_len <= recv.len()`.
- Reach-scan owner matched on identifier boundaries.
- Combined-diff `@@@` headers seed `new_line` from the `+` range correctly.
- Exit code: policy violation exits 1; tool error exits 2.

## Trust boundary

`unsafe-review` 0.3.7 remains advisory static unsafe coverage evidence. It does
not prove memory safety, UB-free status, Miri-clean status, site execution,
calibrated precision/recall, or policy readiness. The telemetry is a
usage/subset signal — not a precision or recall metric. The GitHub Action is
advisory; it does not post comments, block merges, or execute witnesses by
default. unsafe-review does not run witnesses, post comments, edit source, or
block by default.
