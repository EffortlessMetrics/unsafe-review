# 0.3.3 Bun Manual-Candidate Cockpit Usability Patch

`unsafe-review` 0.3.3 is a Bun dogfood / manual-candidate / repo cockpit
usability patch. It publishes the history-preserved source import of the
post-0.3.2 swarm usability lane.

The release goal is narrow:

- frame every static finding as a hypothesis pending external confirmation,
  with a concrete per-card confirmation cue,
- make Bun-oriented manual candidates carry proof mode, stable-byte metadata,
  and JS/TypeScript oracle mapping,
- project manual candidates into a usable implementer handoff: manual repair
  sidecar, review-kit mix summary, repair-queue cockpit panel, and tokmd
  packet inputs,
- keep dogfood reports verifier-aligned with committed manual-candidate
  examples,
- preserve the existing no-witness-execution, no-commenting, no-source-editing,
  and no-default-blocking boundaries.

## Highlights

- ReviewCards carry confirmation cues: `hypothesis_to_confirm`,
  `build_this_first`, `minimal_repro`, and `confirmation_step` are projected
  into `cards.json`, comment plans, agent context packets, and terminal
  `first-pr` output. Each cue states that unsafe-review did not run it.
- Manual candidates carry stable-byte metadata (`class`, `source`, `sink`,
  `hazard`, `observable`, `proof_required`, `suggested_fix_boundary`,
  `pr_aperture`, `ledger_state`) and oracle maps (`rust_seam`,
  `oracle_language`, `oracle_path`, `oracle_kind`) with required node-parity
  coverage for Bun-oriented candidates.
- ReviewCard proof paths are projected across JSON, Markdown, comment-plan,
  witness-plan, and outcome outputs.
- `first-pr` writes `tokmd-packets.json`: formatting-only manual packet inputs
  with comment budget and preset inputs, recorded with `tokmd_run = false`.
- The manual repair handoff path is complete: manual repair sidecar buckets,
  review-kit manual-candidate mix summary, repair-queue cockpit panel with
  agent-readiness cues, and repair-queue bucket reasons in summaries.
- Summaries report diff scope (`changed_files`, `changed_rust_files`,
  `changed_non_rust_files`); `repo --list-files` supports JSON and Markdown
  output with recorded scan scope.
- A dogfood drift guard requires the Bun manual-candidate smoke report to list
  every committed manual-candidate example ID and primary file:line.
- The ReviewCard trust boundary is unified across output surfaces.
- Stale-span-after-reentry detection and optional confirmation-cue execution
  are recorded as known next analyzer work, not implemented behavior.

## Versioning

The public crates move together for this patch:

```text
unsafe-review-core 0.3.2 -> 0.3.3
unsafe-review-cli  0.3.2 -> 0.3.3
unsafe-review      0.3.2 -> 0.3.3
```

This follows the synchronized public crate release pattern used by earlier
patch releases.

## Preconditions

Before publication:

- source `main` must contain source PR
  <https://github.com/EffortlessMetrics/unsafe-review/pull/526>,
- source merge commit `345c5b1a` must be reachable from source `main`,
- imported swarm commit `4b444594` must be reachable from source history,
- no `0.3.2` release tag is rewritten,
- no `0.3.3` crate is published before dry-runs and package inspection pass.

## Non-goals

0.3.3 is not:

- memory-safety proof,
- UB-free status,
- Miri-clean status,
- site-execution proof,
- calibrated precision/recall,
- policy readiness,
- witness execution,
- automatic commenting,
- automatic source editing,
- default blocking policy,
- analyzer discovery of manual candidates (manual candidates remain
  manual/advisory with `analyzer_discovered = false`),
- automatic confirmation-cue execution (cues are emitted, never run).

## Trust Boundary

`unsafe-review` 0.3.3 reports static unsafe-review evidence and manually
imported advisory candidates. It does not prove memory safety, UB-free status,
Miri-clean status, site execution, witness success, calibrated precision/recall,
or policy readiness.

It does not execute witnesses, post comments, edit source, or enforce blocking
policy by default.
