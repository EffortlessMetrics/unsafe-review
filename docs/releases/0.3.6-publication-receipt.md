## What shipped

- reject unknown CLI flags; `first-pr --out` fails loudly (exit 2) instead of
  silently writing to the default location
  ([#531](https://github.com/EffortlessMetrics/unsafe-review/issues/531))
- `receipt import-sanitizer --allow-runtime` accepts runtime/program ASAN
  witnesses, distinguishing fired (`confirmed`) from clean (`not_reproduced`),
  with `--strength` values discoverable in help
  ([#519](https://github.com/EffortlessMetrics/unsafe-review/issues/519))
- `repo --out` writes a scan-started status stub before analysis so a kill that
  precedes the first pipeline event still leaves a durable record
  ([#518](https://github.com/EffortlessMetrics/unsafe-review/issues/518))
- confirm command provenance + owner-token hardening, repair-queue bounded
  applicable edits, and an additive stable-byte sub-class hint.

## Trust boundary

`unsafe-review` 0.3.6 remains advisory static unsafe coverage evidence. Imported
sanitizer witnesses are evidence metadata about what a witness observed (a fired
sanitizer records an observed failure; a clean run records only that no signal
was observed under that witness), not a proof of overall safety. It does not run
witnesses, post comments, edit source, or block by default.
