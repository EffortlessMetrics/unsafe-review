# 0.3.5 Instrument-Truthfulness Patch

`unsafe-review` provides baseline-aware unsafe evidence coverage artifacts for
repo badges, PR gates, line-comment plans, LSP/LLM context, and ub-review.

0.3.5 is the instrument-truthfulness patch. It publishes the history-preserved
source import of the post-0.3.4 truthfulness lane. It makes the instrument's
core assertions exact for downstream automation — so a maintainer, badge, CI
wrapper, LSP/agent loop, or ub-review can trust what a run says it measured.

The release goal is narrow and adds no analyzer breadth:

- skip nested git checkouts and gitfile worktrees during repo discovery so
  scratch worktrees and vendored copies do not inflate scan counts,
- reject an unparseable `--diff` as an explicit input error instead of silently
  scanning the whole repository under a `scope: "diff"` label,
- treat a valid empty diff as a complete diff-scoped no-op (0 files, 0 cards,
  `no-new-debt` passes) rather than a whole-repo fallback,
- distinguish exit categories so a policy violation (exit 1) is mechanically
  separable from a tool/input failure (exit 2),
- record receipt provenance (tool version, schema version, input identity,
  platform, dirty marker) so an artifact is traceable to the run that produced
  it,
- emit a partial status sidecar with an explicit `stop_reason` when a scan is
  capped by `--max-cards` so a truncated scan is never reported as complete,
- document per-file scan cost and the large-repo scoping workflow.

## What changed

- Repo discovery skips any subdirectory containing a `.git` entry (nested
  checkout or gitfile worktree) by default, independent of gitignore handling.
  All scan surfaces share the same discovery, so the exclusion is uniform.
  (#1552)
- An unparseable `--diff` is an explicit input error (exit 2, no analysis, no
  artifacts). Empty/whitespace-only and binary-only diffs remain accepted.
  (#1516)
- A valid empty diff is a complete diff-scoped no-op: scope stays `diff`, 0
  selected files, 0 cards, `no-new-debt` exits 0. Clean-PR callers that relied
  on the old whole-repo fallback should use `unsafe-review repo`. (#1558)
- Exit codes are a documented contract: 0 ran to completion (clean or advisory),
  1 ran to completion with a `no-new-debt` policy violation, 2 the tool did not
  complete a review. The stderr prefix names the category. (#1518)
- `cards.json` / `--format json` bump `schema_version` `0.1` -> `0.2` and carry
  a `provenance` block (tool version, `generated_at`, resolved root, base/head
  SHAs, diff path + SHA-256, dirty marker). The bump is additive; absent fields
  are omitted, not null. (#1517)
- `repo --max-cards N` capped scans emit a partial status sidecar
  (`partial: true`, `stop_reason: "max_cards"`, `cap: N`) and exit 0. A unified
  `stop_reason` (`none`/`max_cards`/`timeout`/`terminated`/`error`) makes every
  status self-describing; an error is never mislabeled as a timeout. SPEC-0035
  field names and phase vocabulary were reconciled to the shipped sidecar.
  (#1545)
- The CLI guide documents per-file scan cost and the large-repo scoping
  workflow. (#1546)

## Consumer migration

Two changes are breaking for automated consumers (both documented for ub-review
in EffortlessMetrics/ub-review#396):

- exit code: a `no-new-debt` policy violation now exits **1**, not **2**.
  Callers testing `$? -eq 2` for policy failures should test `$? -eq 1`;
  `$? -ne 0` callers are unaffected.
- `cards.json` `schema_version` is now **0.2**; consumers routing on
  `schema_version` should accept `0.2` wherever they accepted `0.1` (additive,
  every prior field unchanged).

## Versioning

The public crates move together for this patch:

```text
unsafe-review-core 0.3.4 -> 0.3.5
unsafe-review-cli  0.3.4 -> 0.3.5
unsafe-review      0.3.4 -> 0.3.5
```

This follows the synchronized public crate release pattern used by earlier
patch releases.

## Preconditions

Before publication:

- source `main` must contain the import merge from this release-prep branch,
- the import merge must preserve swarm PR-squashed history and reach swarm main
  `13a33b31`,
- no existing release tag is rewritten (`v0.3.4` is the latest published tag),
- no `0.3.5` crate is published before dry-runs and package inspection pass.

## Non-goals

0.3.5 is not:

- analyzer expansion or a UB detector improvement,
- memory-safety proof,
- UB-free status,
- Miri-clean status,
- site-execution proof,
- calibrated precision/recall,
- policy readiness,
- witness execution,
- automatic commenting,
- automatic source editing,
- default blocking policy.

## Trust Boundary

`unsafe-review` 0.3.5 reports static unsafe-review evidence and baseline
coverage artifacts. It does not prove memory safety, UB-free status, Miri-clean
status, site execution, witness success, calibrated precision/recall, or policy
readiness.

It does not execute witnesses, post comments, edit source, or enforce blocking
policy by default.
