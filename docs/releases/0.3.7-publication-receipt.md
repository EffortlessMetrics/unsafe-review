### Publish method note (new for 0.3.7)

0.3.7 is the first release published via the `crates-publish` GitHub Actions
workflow (`.github/workflows/crates-publish.yml`) on
`EffortlessMetrics/unsafe-review`. The workflow dispatches from source `main`,
verifies all three crate versions match the `version` input, publishes
`unsafe-review-core` first, waits for the sparse index to expose it (retry loop,
up to 10 attempts with 15 s sleep), then publishes `unsafe-review-cli`, then
`unsafe-review`. The org `CARGO_REGISTRY_TOKEN` secret was granted to
`EffortlessMetrics/unsafe-review` as a prerequisite (one-time org-admin step).

## Trust boundary

`unsafe-review` 0.3.7 remains advisory static unsafe coverage evidence. It does
not prove memory safety, UB-free status, Miri-clean status, site execution,
calibrated precision/recall, or policy readiness. The telemetry is a
usage/subset signal — not a precision or recall metric. The GitHub Action is
advisory; it does not post comments, block merges, or execute witnesses by
default. unsafe-review does not run witnesses, post comments, edit source, or
block by default.
