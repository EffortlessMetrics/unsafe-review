# 0.3.6 CLI-Truthfulness and Evidence-Intake Patch

`unsafe-review` provides baseline-aware unsafe evidence coverage artifacts for
repo badges, PR gates, line-comment plans, LSP/LLM context, and ub-review.

0.3.6 is the CLI-truthfulness and evidence-intake patch. It publishes the
history-preserved source import of the post-0.3.5 lane. It makes the command
line hard to misuse and lets real external evidence enter the receipt ledger —
so a maintainer can install it, run it, and trust it.

The release goal is narrow and adds no analyzer breadth:

- reject unknown CLI flags instead of silently ignoring them, so a mistyped or
  misplaced flag fails loudly rather than dropping user intent on the floor,
- accept runtime/program sanitizer (AddressSanitizer and similar) witness logs
  as receipt evidence, with an honest passing-vs-observed-failure distinction,
- make the accepted `receipt --strength` values discoverable in CLI help,
- write a scan-started status stub for `repo --out` runs so a run killed before
  it finishes still leaves a durable, honest record,
- surface `confirm` command provenance and harden owner-derived argv tokens,
- carry a bounded `applicable_edit` hint on repairable repair-queue entries for
  consumers, without ever editing source,
- add an additive `stable_byte_sub_class` hint on stable-byte-source auto-cards.

## What changed

- `first-pr --out ...` (and other subcommands) no longer silently ignore an
  unknown flag. The run now exits 2 with a diagnostic that names the flag and
  suggests `--out-dir`, and writes no review kit to the default location.
  ([#531](https://github.com/EffortlessMetrics/unsafe-review/issues/531))
- `receipt import-sanitizer --allow-runtime` accepts a runtime/program
  AddressSanitizer witness log without requiring a `test result: ok` line. A run
  where the sanitizer fires records verdict `confirmed` (a failure was observed,
  not a safety claim); a clean run records `not_reproduced` (no signal this run,
  not a safety claim). Default behavior (requiring `test result: ok`) is
  unchanged; runtime intake is opt-in.
  ([#519](https://github.com/EffortlessMetrics/unsafe-review/issues/519))
- The accepted `receipt --strength` values
  (`configured`/`ran`/`test_targeted`/`site_reached`/`reviewed`) are now listed
  in `receipt template` help and in the unknown-value error.
  ([#519](https://github.com/EffortlessMetrics/unsafe-review/issues/519))
- `repo --out <path>` writes a "scan started" status stub to
  `<path>.status.json` before analysis begins, so a process killed before it can
  finish still leaves a durable record (`phase: "discovering"`,
  `completed: false`). Every normal, capped, timed-out, or interrupted outcome
  supersedes the stub.
  ([#518](https://github.com/EffortlessMetrics/unsafe-review/issues/518))
- `confirm` surfaces command provenance (analyzer-derived route vs `--command`
  override) and validates owner-derived argv tokens, so a flag-shaped or
  shell-ish owner can never be spliced into a routed witness command; an invalid
  owner yields no command rather than a fabricated one. `runtime_executed` is
  never fabricated.
- Repair-queue entries carry a bounded `applicable_edit` hint on repairable
  buckets so consumers can render one-click suggestions; terminal
  `do_not_auto_repair` / `requires_human_review` entries carry no such hint.
  unsafe-review does not edit source.
- Stable-byte-source auto-cards carry an additive `stable_byte_sub_class` hint
  (omitted for other families) — a heuristic aperture label, not a claim.

## Trust boundary

`unsafe-review` 0.3.6 remains advisory static unsafe coverage evidence. It does
not prove memory safety, UB-free status, Miri-clean status, site execution,
calibrated precision/recall, or policy readiness. An imported sanitizer witness
is evidence metadata about what a witness observed, not a proof of overall
safety: a fired sanitizer records an observed failure, and a clean run records
only that no signal was observed under that witness. unsafe-review does not run
witnesses, post comments, edit source, or block by default.
