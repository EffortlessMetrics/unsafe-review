# 0.3.1 Crates.io README Image Hotfix

`unsafe-review` 0.3.1 is a package-surface patch after source history repair.

The release goal is narrow:

- publish crates that are built from source history containing the reviewed
  swarm commit graph,
- ensure the facade crate README uses the stable raw GitHub logo URL on
  crates.io,
- preserve the 0.3.0 advisory review cockpit behavior and trust boundary.

The package-surface fix is:

```text
crates/unsafe-review/README.md
```

using:

```text
https://raw.githubusercontent.com/EffortlessMetrics/unsafe-review/main/unsafe-review-logo.svg
```

## Non-goals

0.3.1 is not:

- a new analyzer release,
- a Bun investigation lane,
- a calibration release,
- a policy-readiness release,
- a witness execution release,
- an automatic comment release,
- an automatic source-editing release.

## Versioning

The public crates move together for this patch:

```text
unsafe-review-core 0.3.0 -> 0.3.1
unsafe-review-cli  0.3.0 -> 0.3.1
unsafe-review      0.3.0 -> 0.3.1
```

This follows the existing synchronized public crate release pattern. The
facade README is the package-visible fix, but the synchronized version bump
keeps installed facade, CLI, and core metadata aligned.

## Preconditions

Before publication:

- source `main` must contain the history-preserving swarm catch-up merge,
- swarm commit `3a71a65c` must be reachable from source history,
- `crates/unsafe-review/README.md` must contain the stable raw GitHub logo URL,
- no `0.3.0` release tag is rewritten,
- no 0.3.1 crate is published before dry-runs and package inspection pass.

## Trust Boundary

`unsafe-review` 0.3.1 reports static unsafe-review evidence. It does not prove
memory safety, UB-free status, Miri-clean status, site execution, witness
success, calibrated precision/recall, or policy readiness.

It does not execute witnesses, post comments, edit source, or enforce blocking
policy by default.
