# 0.3.0 Advisory Review Cockpit Target

Date: 2026-05-31

`unsafe-review` 0.3.0 is the advisory review cockpit release. The goal is not
broader analyzer breadth, calibrated policy gating, witness execution, or source
edits. The goal is that a Rust maintainer can install the tool, run one PR
command, open one cockpit summary, inspect the top ReviewCards, and hand one
bounded card packet to a human or agent without overtrusting the result.

The release target is:

```text
cargo install unsafe-review --locked
unsafe-review doctor
unsafe-review first-pr --base origin/main
open target/unsafe-review/pr-summary.md
unsafe-review explain <card-id>
unsafe-review context <card-id> --json
```

## Product Boundary

0.3.0 remains static unsafe contract review evidence. It is not:

- memory-safety proof
- UB-free status
- Miri-clean status
- site-execution proof
- calibrated precision or recall
- policy readiness
- default policy gating
- automatic comments
- source edits
- witness execution
- agent execution

Every user-facing surface must preserve the trust boundary: `unsafe-review`
finds unsafe Rust changes missing safety contract, guard, test, or witness
evidence. It routes reviewers to concrete next actions; it does not prove the
repository safe.

## Release Shape

The 0.3.0 user path centers on:

| Surface | 0.3.0 target |
|---|---|
| `first-pr` / `review` | One-command advisory review kit for PR review. Writes the manifest, cards, summaries, SARIF, comment plan, witness plan, saved LSP JSON, and repair queue. |
| `review-kit.json` | Manifest that lists emitted artifacts, summary counts, top card handoff, and trust boundary. |
| `pr-summary.md` | Reviewer cockpit front panel: top card, missing evidence, next action, witness route, explain/context commands, and trust boundary. |
| `github-summary.md` | Bounded CI doorway that points to the full review kit without duplicating the report. |
| `comment-plan.json` | Artifact-only review budget with selected and not-selected reasons. Comments are planned, not posted. |
| `witness-plan.md` | Reviewer-readable route plan grouped by witness family with limitations and receipt hints. |
| `lsp.json` | Saved read-only editor projection. No live LSP claim. |
| `repair-queue.json` | Copy-only human/agent handoff buckets projected from ReviewCards. No agent execution, source edits, suppression, or repair-success claim. |
| `explain <card-id>` | Human-readable card explanation. |
| `context <card-id> --json` | Bounded agent/human card packet. |
| Artifact verifier | Checks the first-pr review kit for parseability, artifact identity, card identity consistency, trust-boundary wording, and absence of safety/proof overclaims. |

## Required Work

0.3.0 should be prepared by narrow source PRs:

1. Promote the advisory review kit artifacts: `review-kit.json`,
   `github-summary.md`, PR summary cockpit handoff, terminal handoff, and
   verifier updates.
2. Promote comment-budget, repair-queue, and receipt-audit surfaces if they
   remain ReviewCard projections and preserve the advisory boundary.
3. Promote only curated fixture-pinned analyzer/card-correctness controls needed
   for 0.3.0 dogfood or Bun investigation.
4. Prepare version bumps, package dry-runs, install smoke, first-pr smoke,
   artifact verifier smoke, explain/context smoke, doctor smoke, and support
   smoke.

Analyzer/evidence changes still start in `unsafe-review-swarm` by default.
Source receives curated promotions, release/public surface work, and urgent
hotfixes only.

## Deferred Past 0.3.0

These are intentionally not part of the 0.3.0 cockpit release:

- calibrated precision or recall claims
- policy-eligible support claims
- default blocking CI
- automatic inline comments
- automatic source edits
- witness execution by default
- broad analyzer expansion without dogfood or fixture controls
- Bun vulnerability claims in release notes

Bun investigation is a post-release use case. The release provides the advisory
review kit; any Bun finding still requires manual proof of a safe API path,
local prevention, and a regression or witness appropriate to that finding.

## Done Criteria

The 0.3.0 lane is ready for release when:

- `cargo install unsafe-review --locked` works for the published graph.
- `unsafe-review doctor` is useful and preserves the trust boundary.
- `unsafe-review first-pr --base origin/main` writes the standard advisory
  review kit.
- `review-kit.json` lists every emitted artifact and top-card handoff.
- `pr-summary.md` is useful as the reviewer cockpit.
- `github-summary.md` stays bounded and points to the full review kit.
- `comment-plan.json` remains artifact-only, capped, and explicit about why
  cards were or were not selected for inline review.
- repair queue and receipt-audit surfaces, if promoted, remain advisory
  ReviewCard projections.
- `unsafe-review explain <card-id>` and `unsafe-review context <card-id> --json`
  are readable, scoped, and non-overclaiming.
- no-card output is honest and does not say or imply "all clear".
- bundle verification rejects missing artifacts and safety/proof wording.
- release notes describe the advisory review cockpit, not proof or policy
  authority.
