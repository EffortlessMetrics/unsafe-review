# 0.3.8 Corpus Control Plane and Surfacing Correctness

`unsafe-review` provides baseline-aware unsafe evidence coverage artifacts for
repo badges, PR gates, line-comment plans, LSP/LLM context, and ub-review.

0.3.8 is the corpus control-plane and declaration/surfacing correctness patch.
It turns the recent detector, stance, corpus, and surface-determinism rails into
checked release infrastructure. It also makes unsafe fn/trait declarations
classify as `unsafe_declaration` and separates inline-comment surfacing policy
from operation-family labels.

## What changed

### Declaration and surfacing correctness

- Unsafe fn and unsafe trait declarations now use the additive
  `unsafe_declaration` operation family regardless of documentation state.
  Unsafe impls and genuine unknown unsafe blocks remain unchanged.
  ([#1746](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1746))
- Comment eligibility now comes from explicit surfacing disposition, not from
  `OperationFamily::Unknown`. Changing a family label alone cannot make a card
  eligible for inline comments.
  ([#1747](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1747))
- The residual unknown report records the remaining unknown shape by site kind,
  syntax form, crate, and fixture comment-plan surfacing. The report chooses no
  next classifier family from count reduction alone; unsafe impl classification
  remains a separate stance/corpus lane.
  ([#1751](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1751))

### Corpus and control-plane rails

- Detector contracts, stance decisions, and spec coverage now have checked
  policy ledgers. The gates moved from informational to enforcing once current
  exceptions were closed or explicitly receipted.
  ([#1723](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1723),
  [#1725](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1725),

  [#1728](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1728),

  [#1732](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1732),
  [#1734](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1734))
- The pinned real-repo corpus now has an executable `dogfood-exec` runner with
  bounded per-target timeout support, plus additional nix, simdutf8, and
  zerocopy targets and a release-readiness checklist for the heavy corpus pass.
  ([#1737](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1737),
  [#1738](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1738),

  [#1744](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1744))
- The real-PR movement corpus and stance-coverage map give PR-shaped movement
  and stance decisions deterministic checks without turning corpus telemetry
  into calibrated accuracy claims.
  ([#1740](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1740),
  [#1741](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1741))

### Deterministic projection surfaces

- Canonical exemplar surfaces now carry committed JSON goldens for
  comment-plan, LSP, and repair-queue projections. `check-fixture-surface-parity`
  keeps those surfaces present for the selected exemplars.
  ([#1739](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1739))
- `check-surface-determinism` rerenders canonical structured outputs multiple
  times and byte-compares the stable surfaces. Comment-plan ordering now has a
  deterministic card-id tie-breaker.
  ([#1742](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1742),
  [#1749](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1749))

### Tooling hygiene

- `xtask` resolves the workspace root at runtime via `--workspace-root`,
  `UNSAFE_REVIEW_WORKSPACE_ROOT`, or a validated repository/current-directory
  fallback. Reusing a cached xtask binary from a shared Cargo target directory
  can no longer make it read policy from a stale checkout.
  ([#1748](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1748))
- Agent orchestration docs now record the cached-binary root failure mode and
  every-PR golden determinism requirements as enforced rails, not just lessons.
  ([#1750](https://github.com/EffortlessMetrics/unsafe-review-swarm/pull/1750))

## Trust boundary

`unsafe-review` 0.3.8 remains advisory static unsafe coverage evidence. It does
not prove memory safety, UB-free status, Miri-clean status, site execution,
calibrated precision/recall, or policy readiness. The corpus and real-PR checks
are diagnostics and release-readiness controls, not accuracy measurements or
merge verdicts. unsafe-review does not run witnesses, post comments, edit
source, or block by default.
