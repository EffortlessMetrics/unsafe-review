## What shipped (the coverage-instrument usability batch)

unsafe-review is now the baseline-aware unsafe evidence coverage instrument:
coverage-slot model (SPEC-0029), baseline coverage movement with diff-scoped
`--policy no-new-debt` (SPEC-0030, the adoption keystone), `baseline init`/`add`
with a coverage snapshot that activates `worsened`, baseline-aware numeric badges
(SPEC-0031), comment-plan coverage-gap anchoring + forbidden-claim checks
(SPEC-0032), the file:range LLM/agent context packet (SPEC-0033), the
`unsafe-review-gate.json` routing manifest for ub-review (SPEC-0034), repo-scan
diagnosability (SPEC-0035), manual candidate `new`/`lint`/`import`, and a narrow
stable-byte coverage detector v1 (getter-reentry missing-evidence).

## Honest notes / known limits

- `unsafe-review-gate.json` (schema `unsafe-review-gate/v1`) is evidence-only by
  design; the deterministic floor wall-time is captured by the CI gate workflow,
  not the manifest (ub-review composes it; tracked in EffortlessMetrics/ub-review
  #336).
- The stable-byte detector v1 is deliberately narrow (JS-backed byte/span capture
  -> JS reentry -> use without refetch/snapshot/pin). It emits "missing
  stable-byte evidence" coverage gaps, never a UB verdict. RAB/SAB/native-FFI
  breadth is out of scope.

## Trust boundary

`unsafe-review` 0.3.4 remains advisory static unsafe coverage evidence. No proof,
UB-free, Miri-clean, site-execution, or calibrated precision/recall claims. No
posting by default, no default blocking, no witness execution by default. Manual
candidates remain manual/advisory; confirmation cues are emitted, never executed.
