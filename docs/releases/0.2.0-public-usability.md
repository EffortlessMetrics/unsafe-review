# 0.2.0 Public Usability Target

Date: 2026-05-20

`unsafe-review` 0.2.0 is the public usability release. The goal is not broader
analysis, policy authority, or live editor integration. The goal is that a Rust
maintainer can install the tool, run one PR command, open one summary, explain
one card, and take one concrete review action without overtrusting the result.

The release target is:

```text
cargo install unsafe-review --locked
unsafe-review doctor
unsafe-review first-pr --base origin/main
open target/unsafe-review/pr-summary.md
unsafe-review explain <card-id>
```

## Product Boundary

0.2.0 remains static unsafe contract review evidence. It is not:

- memory-safety proof
- UB-free status
- Miri-clean status
- site-execution proof
- calibrated precision or recall
- default policy gating
- automatic comments
- source edits
- witness execution
- live LSP or VS Code extension support

Every user-facing surface must preserve the trust boundary: `unsafe-review`
finds unsafe Rust changes missing a safety contract, guard, test, or witness.
It routes reviewers to the cheapest credible next action; it does not prove the
repository safe.

## Release Shape

The 0.2.0 user path centers on:

| Surface | 0.2.0 target |
|---|---|
| `doctor` | First-run readiness check that reports git, workspace, artifact directory, optional witness tools, and the trust boundary. Missing witness tools are informational. |
| `first-pr` / `review` | One-command advisory bundle for PR review. Writes cards, PR summary, SARIF, comment plan, witness plan, and saved LSP JSON. |
| `pr-summary.md` | Reviewer front panel: changed unsafe seams, top card, missing evidence, witness route, and trust boundary. |
| `explain <card-id>` | Reviewer-first card explanation: why it exists, required conditions, found evidence, missing evidence, resolution guidance, non-resolution examples, witness route, and trust boundary. |
| `witness-plan.md` | Reviewer-readable route plan grouped by witness family with limitations and receipt hints. |
| `comment-plan.json` | Artifact-only, capped, changed-line-oriented comment suggestions. No posting by default. |
| `support` | Plain support posture command that says what is experimental, advisory, deferred, or not default. |
| Artifact verifier | Checks the first-pr bundle for parseability, card identity consistency, trust-boundary wording, comment-plan caps, route limitations, and absence of safety/proof overclaims. |

## Required Work

0.2.0 should be prepared by narrow source PRs:

1. Verify the complete `first-pr` artifact bundle.
2. Polish the `first-pr` terminal summary.
3. Standardize no-card advisory wording across public surfaces.
4. Make `explain` consistently reviewer-first.
5. Add explain examples for common card families.
6. Make `doctor` a first-run readiness check.
7. Keep `unsafe-review support` aligned with the current support posture.
8. Keep witness-plan grouping, route limitations, and receipt hints aligned with
   the current receipt commands.
9. Add receipt-template authoring if it remains saved-output-only.
10. Explain advisory policy report classifications and pin the policy report JSON schema.
11. Add dogfood usefulness notes for 0.2.0 targets.
12. Prepare release readiness with package lists, dry-runs, install smoke, first-pr smoke, explain smoke, doctor smoke, and support smoke.

Analyzer/evidence changes still start in `unsafe-review-swarm` by default.
Source receives curated promotions, release/public surface work, and urgent
hotfixes only.

## Deferred Past 0.2.0

These are intentionally not part of the 0.2.0 public usability release:

- default blocking CI
- automatic inline comments
- automatic source edits
- witness execution by default
- live stdio LSP support as a source release claim
- VS Code or Open VSX publication
- broad suppressions
- calibrated precision or recall claims
- support-tier promotion without fixture, dogfood, receipt, schema, or release proof

Saved `lsp.json` remains the 0.2.0 editor-adjacent artifact. Live LSP work may
continue in swarm as a later editor-lane seed, but 0.2.0 should not depend on
that lane.

## Done Criteria

The 0.2.0 lane is ready for release when:

- `cargo install unsafe-review --locked` works for the published graph.
- `unsafe-review doctor` is useful and non-scary for first-time users.
- `unsafe-review first-pr --base origin/main` writes the standard advisory bundle.
- `first-pr` output points at `pr-summary.md` and `unsafe-review explain <card-id>`.
- `unsafe-review explain <card-id>` is readable, obligation-specific, and actionable.
- no-card output is honest and does not say or imply "all clear".
- `comment-plan.json` remains artifact-only and capped.
- `witness-plan.md` gives credible route limits and receipt hints.
- `unsafe-review support` prevents overclaiming.
- badge, README, first-use docs, and support docs match the product boundary.
- bundle verification rejects missing artifacts and safety/proof wording.
- release notes describe public usability, not proof or policy authority.
