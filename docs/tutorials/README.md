# Tutorials

Tutorials are learning-oriented, end-to-end paths for a first successful run. They
prioritize a guided result over complete coverage.

Start here when you are new to `unsafe-review` and want to see one review card from
input diff to reviewer action.

## Available tutorials

- [First review card](first-review-card.md): run the bundled fixture, read the
  generated card, and connect the result to the unsafe review workflow.

## Tutorial rules

Tutorials should:

- use copy-pasteable commands from a clean checkout;
- explain only the concepts needed for the next step;
- end with an observable result;
- link to how-to guides for production tasks and to reference docs for complete
  option lists.

Tutorials should not become API references, design essays, or release plans.
