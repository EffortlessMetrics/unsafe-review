# 2026-05-31 - release 0.3.1 preparation

Scope: prepare the `0.3.1` crates.io README/image package-surface hotfix for
`unsafe-review-core`, `unsafe-review-cli`, and `unsafe-review`.

This is a release-preparation receipt, not a publication receipt.

## Release framing

`0.3.1` is a patch release after source history repair. It publishes from source
history that contains the reviewed swarm PR-squashed commits, including the
crates.io-safe README image fix.

The patch is narrow:

- keep the 0.3.0 advisory review cockpit behavior,
- keep the 0.3.0 trust boundary,
- ensure the facade crate README uses the stable raw GitHub logo URL on
  crates.io,
- keep the three public crate versions synchronized at `0.3.1`.

`0.3.1` is not a new analyzer release, Bun lane, calibration release,
policy-readiness release, witness execution release, automatic comment release,
or source-editing release.

## Stop conditions

Do not publish if:

- source history does not contain the reviewed commit trail for the README fix,
- crates.io reports any of the three `0.3.1` versions already published,
- any package list check fails,
- the packaged facade README does not contain the stable raw GitHub logo URL,
- staged downstream dry-runs fail after their upstream `0.3.1` crate has
  propagated on crates.io,
- `first-pr` does not emit the full review kit,
- `check-first-pr-artifacts` does not verify the full review kit,
- `explain`, `context`, `doctor`, or `support` wording implies safety,
  UB-free status, Miri-clean status, site execution, automatic comments, source
  edits, policy readiness, calibrated precision, or calibrated recall.

## Trust boundary

`unsafe-review` `0.3.1` reports static unsafe-review evidence. It does not
prove memory safety, UB-free status, Miri-clean status, site execution, witness
success, calibrated precision/recall, or policy readiness.

It does not execute witnesses, post comments, edit source, or enforce blocking
policy by default.
