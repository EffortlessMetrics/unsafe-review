Scope: prepare the synchronized `0.3.5` instrument-truthfulness patch from
source `main` after the history-preserving swarm import on branch
`release/0.3.5-instrument-truthfulness`.

This is release preparation only. It does not publish crates, add analyzer

policy gating.

## What this release carries

Branch `release/0.3.5-instrument-truthfulness` imports swarm main `13a33b31`
(history-preserved `--no-ff` merge, PR-squashed history reachable), covering the
post-0.3.4 instrument-truthfulness lane recorded under `CHANGELOG.md` `0.3.5`:

- nested git checkout / gitfile worktree scan exclusion so scratch worktrees and
  vendored copies do not inflate scan counts (#1552),
- unparseable `--diff` rejected as an explicit input error instead of a silent
  whole-repo fallback under `scope: "diff"` (#1516),
- valid empty diff treated as a complete diff-scoped no-op (#1558),
- documented exit-code contract: 0 ran-clean/advisory, 1 policy violation, 2
  tool did not complete (#1518),
- `cards.json` / `--format json` schema `0.1` -> `0.2` with a `provenance` block
  (tool version, generated_at, root, base/head SHAs, diff path + SHA-256, dirty
  marker), additive (#1517),
- `repo --max-cards` partial status sidecar with a unified `stop_reason`
  (`none`/`max_cards`/`timeout`/`terminated`/`error`); SPEC-0035 field names and
  phase vocabulary reconciled to the shipped sidecar (#1545),
- CLI guide scan-cost and large-repo scoping documentation (#1546).
