# 2026-06-07 - 0.3.4 consumer validation

Scope: validate the released `unsafe-review` 0.3.4 in its two intended consumers
(ub-review integration; the Bun foreign-repo hunt) and record what it proved and
did not prove, with every follow-up filed as a narrow, artifact-backed issue.

This is a validation handoff. It makes no proof, UB-free, Miri-clean,
site-execution, or calibrated precision/recall claims. unsafe-review remains the
advisory unsafe-coverage instrument; ub-review is the LLM review/orchestration
layer on top of it.

## What 0.3.4 proved

1. **ub-review consumes the structured artifacts, not Markdown.** ub-review now
   passes `--out` to `unsafe-review first-pr`, parses `unsafe-review-gate.json`
   routed by `schema_version` (`unsafe-review-gate/v1`), follows the artifact
   pointers to `cards.json` / `comment-plan.json`, and surfaces the structured
   movement summary + coverage findings into its model lanes — no stdout/Markdown
   scraping. EffortlessMetrics/ub-review#359 (impl) -> PR #363 (merged, passed
   ub-review's own recursive-dogfood gate).
2. **ub-review posts inline path:line review comments from the comment-plan.**
   The bounded comment-plan candidates become inline review comments anchored to
   `path:line`, naming the coverage gap + next action + trust boundary, deduped
   and budget-capped. ub-review#360 -> PR #365 (merged). The `suggestion`-block
   facility is wired and tested.
3. **Baseline / no-new-debt works on a real foreign brownfield repo.** On
   EffortlessSteven/bun: `baseline init` recorded pre-existing debt; a re-scan
   with `--policy no-new-debt` over 59 inherited gaps **passed** (exit 0), and an
   injected new gap **failed** (exit 2, `1 new gap, 0 worsened`). Card IDs stable
   across scans; movement deterministic.
4. **The architecture boundary holds in practice.** unsafe-review emits the
   bounded optic + comment plan; ub-review ingests artifacts and owns
   orchestration/posting. The deterministic floor stays the hard gate; the LLM
   lanes are advisory. No overclaim crossed the boundary in either consumer.

## What 0.3.4 did not prove / friction surfaced (all filed)

- **Full one-click suggestions are not yet possible.** `repair-queue/0.1`
  provides classification + prose guidance but no applicable-edit field, so
  ub-review cannot render GitHub `suggestion` blocks without fabricating edits
  (which it correctly refused to do). Filed: unsafe-review-swarm#1542.
- **`baseline init --out` still writes the snapshot into `--root/policy/`** —
  a real side-effect that dirties a read-only consumer repo. Filed: #1543.
- **stable-byte v1 is too narrow as an auto-detector on JS-runtime code.** The
  canonical Bun seam (`src/runtime/webcore/ByteStream.rs:466` — a JS-owned
  buffer raw fat-pointer held across a JS promise resolution) produces **0
  cards** because it lives in safe Rust above the unsafe-block level; auto-cards
  that do fire on JS-byte sites (`jsc/array_buffer.rs:306`, `Blob.rs:409`,
  `FFIObject.rs:296`) carry no stable-byte sub-class hint. Filed: #1544.
- **`--max-cards` caps silently** (no status artifact, unlike `--timeout-seconds`)
  — a caller cannot tell a complete scan from a truncated one. Filed: #1545.
- **Per-file scan cost is high and undocumented** at Bun scale (28-36s for
  700-1100-line files; full `src/jsc/**` intractable without scoping). Filed:
  #1546.

## Boundary / next lane

Consumer evidence now exists. The next unsafe-review feature lane should be
chosen from these filed issues, not guessed: #1542 (applicable edits) directly
unblocks the full inline-suggestion loop in ub-review; #1544 is the stable-byte
breadth question (a separate lane, not 0.3.x churn); #1543 is a fast bug fix.
The workflow-side floor-time/cost telemetry stays in the ub-review / workflow
layer (ub-review#336), not in `unsafe-review-gate.json`. No broad analyzer
expansion until one of these is picked deliberately.
