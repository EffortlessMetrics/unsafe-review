# LSP and agent projection v0.3 receipt

Date: 2026-05-18
Status: initial experimental saved-projection slice landed
Owner: CLI/core/product

## What landed

The first 0.3 projection slices now project existing `ReviewCard`s into local,
read-only editor and agent surfaces. These surfaces do not create new analyzer
truth and do not edit source.

Merged PRs:

- `#130 lsp: add saved review-card projection`
- `#131 agent: harden bounded packet projection`

The saved LSP projection is generated by:

```bash
unsafe-review check --base origin/main --format lsp --out target/unsafe-review/lsp.json
```

It contains read-only diagnostics, hovers, and copy-command action data derived
from the same cards used by JSON, PR summaries, SARIF, and comment plans.

The bounded agent packet is generated by:

```bash
unsafe-review context <card-id> --json
```

It contains a single-card task, card identity, unsafe site context, required
safety conditions, obligation-level evidence, missing evidence, witness routes,
verify commands, do-not-do rules, stop conditions, and the static-review trust
boundary. Compatibility fields from the initial context-packet scaffold remain:
`card_id`, `required_safety_conditions`, `missing`, and string-array
`allowed_repairs`.

## Current support posture

The saved projection surfaces are experimental. They are suitable for local
dogfood and downstream adapter work, not for policy enforcement or automated
repair.

The repo may claim:

- `--format lsp` emits read-only saved diagnostics, hovers, and copy-command
  action data from `ReviewCard`s.
- `context <card-id> --json` emits a bounded, copy-only agent packet from one
  `ReviewCard`.
- Both surfaces carry the static-review trust boundary.
- Agent packets constrain repair scope; they do not perform repair.

The repo must not claim:

- an LSP server or editor extension exists
- source edits or automatic unsafe rewrites
- automatic agent execution
- automatic PR comments
- Miri, sanitizer, Loom, Kani, or Crux success without imported receipts
- memory-safety proof, UB-free status, or proof that a static test mention
  executed the unsafe site

## Known limits

- There is no LSP server or editor integration yet.
- Code actions are command data only.
- Agent packets are copy-only and still require human review.
- Agent packets use current `ReviewCard` evidence; false positives and false
  negatives in cards propagate into packets.
- There is no receipt import, baseline, suppression, repo badge, or policy gate.

## Next useful work

Prefer dogfood-driven hardening before adding more product surface:

- verify saved LSP JSON shape against a small adapter or editor prototype
- inspect copied agent packets on real unsafe-review cards
- tighten packet wording when a real repair task is ambiguous
- keep all projection fields sourced from `ReviewCard`

Defer:

- LSP server implementation
- source edit actions
- automated repair agents
- repo badge promotion
- baselines or no-new-debt policy
- witness receipt import
- blocking CI policy
