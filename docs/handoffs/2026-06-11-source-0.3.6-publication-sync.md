## Why swarm was briefly behind source

Release preparation and publication are source-side work by policy
(`docs/contributing/SWARM_TO_MAIN.md` routing). During publication, source
gained the catch-up import + release-prep merge (#534) and the receipt (#535)
that swarm did not have. This mirror restores parity and the checkpoint records
that the divergence is acknowledged and absorbed. See
`docs/contributing/SWARM_MIRROR.md` for the mirror-back runbook.

## What shipped (the CLI-truthfulness / evidence-intake batch)

unsafe-review 0.3.6 makes the command line hard to misuse and lets real external
evidence enter the receipt ledger, with no analyzer breadth: reject unknown CLI
flags so `first-pr --out` fails loudly (#531), `receipt import-sanitizer
--allow-runtime` accepts runtime/program ASAN witnesses with a fired-vs-clean
distinction and discoverable `--strength` values (#519), `repo --out` writes a
scan-started status stub before analysis (#518), plus confirm command
provenance + owner-token hardening, repair-queue bounded applicable edits, and a
stable-byte sub-class hint.

## Trust boundary

`unsafe-review` 0.3.6 remains advisory static unsafe coverage evidence. No
proof, UB-free, Miri-clean, site-execution, or calibrated precision/recall
claims. Imported sanitizer witnesses are evidence metadata about what a witness
observed, not safety proofs. No witness execution, comments, source edits, or
default blocking.
