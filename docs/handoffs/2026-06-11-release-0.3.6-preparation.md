## What this release carries

Branch `release/0.3.6-cli-truthfulness` imports swarm main `4dc2f30d`
(history-preserved `--no-ff` merge, PR-squashed history reachable), covering the
post-0.3.5 CLI-truthfulness / evidence-intake lane recorded under `CHANGELOG.md`
`0.3.6`:

- reject unknown CLI flags instead of silently ignoring them; `first-pr --out`
  now exits 2 with a `--out-dir` suggestion and writes no bundle (#531),
- `receipt import-sanitizer --allow-runtime` accepts runtime/program ASAN
  witnesses, distinguishing fired (`confirmed`) from clean (`not_reproduced`),
  with `--strength` values discoverable in help (#519),
- `repo --out` writes a scan-started status stub before analysis so a kill that
  precedes the first pipeline event still leaves a durable record (#518),
- `confirm` command provenance + owner-token hardening (swarm #1514),
- repair-queue bounded `applicable_edit` hints for consumers (swarm #1542),
- additive `stable_byte_sub_class` hint on stable-byte-source auto-cards
  (swarm #1571).
