## Release framing

`0.3.0` is the advisory review cockpit release:

- `first-pr` writes the standard review kit manifest and artifacts,
- `pr-summary.md` is the reviewer cockpit front panel,
- `github-summary.md` is a bounded CI doorway,
- `comment-plan.json` carries review-budget metadata,
- `repair-queue.json` groups card-sized human/agent handoff work,
- `receipt-audit.md` is emitted as saved-receipt metadata context,
- `explain <card-id>` and `context <card-id> --json` keep the card handoff
  discoverable.

`0.3.0` remains an **experimental advisory** release:

- no memory-safety proof,
- no UB-free claim,
- no Miri-clean claim,
- no site-execution claim,
- no witness execution by default,
- no automatic comments,
- no source edits,
- no default blocking policy,
- no calibrated precision or recall claim,
- no policy-readiness claim.

Bun investigation is a post-release use case. The release provides the advisory
review kit; any Bun candidate still needs separate manual proof of a safe API
path, local prevention, and a regression or witness appropriate to that finding.

## Stop conditions

Do not publish if:

- crates.io reports any of the three `0.3.0` versions already published,
- the working tree is dirty,
- any package list check fails,
- the core publish dry-run fails,
- staged downstream dry-runs fail after their upstream `0.3.0` crate has
  propagated on crates.io,
- `first-pr` does not emit the full review kit,
- `check-first-pr-artifacts` does not verify the full review kit,
- `explain`, `context`, `doctor`, or `support` wording implies safety,
  UB-free status, Miri-clean status, site execution, automatic comments, source
  edits, policy readiness, calibrated precision, or calibrated recall.
