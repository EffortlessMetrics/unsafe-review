## Release framing

`0.2.1` is a patch release for public presentation and package-visible badge
semantics:

- the `unsafe-review` badge reports the numeric open actionable gap count,
- the `unsafe-review+` badge reports the numeric missing-or-weak evidence
  count,
- the facade crate README serves as the CLI entry point for crates.io users,
- the root README release badge points to the latest GitHub Release URL.

`0.2.1` does not change analyzer behavior, support tiers, witness handling,
policy behavior, or source-edit behavior.

`0.2.1` remains an **experimental advisory** release:

- no memory-safety proof,
- no UB-free claim,
- no Miri-clean claim,
- no site-execution claim,
- no calibrated precision or recall claim,
- no witness execution by default,
- no automatic comments,
- no source edits,
- no default blocking policy.

The source README intentionally does not add the swarm coverage badge in this
release. Coverage remains a separate source posture decision.

## Stop conditions

Do not publish if:

- crates.io reports any of the three `0.2.1` versions already published,
- the working tree is dirty,
- any package list check fails,
- the core publish dry-run fails,
- the staged downstream dry-runs fail after their upstream `0.2.1` crate has
  propagated on crates.io,
- badge smoke does not emit numeric `message` fields for both endpoint JSON
  files,
- README, badge endpoint, support-tier, first-run, LSP, agent-packet, receipt,
  policy, or release wording implies safety, UB-free status, Miri-clean status,
  site execution, automatic comments, source edits, policy readiness,
  calibrated precision, or calibrated recall.
