## Expected staged dry-run limits

The downstream dry-runs are expected to fail until the dependency crates are
published to crates.io in order:

```text
cargo publish -p unsafe-review-cli --dry-run
  fails because crates.io only has unsafe-review-core 0.1.0

cargo publish -p unsafe-review --dry-run
  fails because crates.io only has unsafe-review-cli 0.1.0
```

This is the expected topological publish constraint for the three-crate graph,
not a packaging regression.

## Manual publish order

Do not publish from automation for this release. Publish manually in dependency
order:

```bash
cargo publish -p unsafe-review-core
cargo publish -p unsafe-review-cli --dry-run
cargo publish -p unsafe-review-cli
cargo publish -p unsafe-review --dry-run
cargo publish -p unsafe-review
```

After crates.io propagation and install smoke, record a separate publication
receipt and then tag `v0.1.1`.

## Stop conditions

Do not publish if:

- crates.io reports any of the three `0.1.1` versions already published,
- the working tree is dirty,
- `cargo publish -p unsafe-review-core --dry-run` fails,
- any downstream dry-run fails for a reason other than the expected unpublished
  dependency version,
- README, badge endpoint, or support-tier wording implies safety, UB-free
  status, Miri-clean status, site execution, or policy readiness.
