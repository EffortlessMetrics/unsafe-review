# 2026-05-31 - release 0.3.2 preparation

Scope: prepare the `0.3.2` repo-dogfood usability patch for
`unsafe-review-core`, `unsafe-review-cli`, and `unsafe-review`.

This is a release-preparation receipt, not a publication receipt.

## Release framing

`0.3.2` is a pre-1.0 patch release after the history-preserved source import of
the swarm repo-dogfood batch.

The patch is focused on repo-mode usability and dogfood handoff quality:

- repo-specific help and large-repo guidance,
- repo include/exclude/gitignore/max-files/list-files selection controls,
- markdown `path:line` locations,
- repo progress/status and partial-output artifacts,
- manual candidate import and projection through explain/context/witness-plan,
- report-only related-sink clusters,
- advisory JS getter reentry after JS-backed descriptor capture detection.

`0.3.2` is not memory-safety proof, UB-free status, Miri-clean status,
site-execution proof, calibrated precision/recall, policy readiness, witness
execution, automatic comment posting, automatic source editing, or default
blocking policy.

## Trust boundary

`unsafe-review` `0.3.2` reports static unsafe-review evidence and manually
imported advisory candidates. It does not prove memory safety, UB-free status,
Miri-clean status, site execution, witness success, calibrated precision/recall,
or policy readiness.

It does not execute witnesses, post comments, edit source, or enforce blocking
policy by default.
